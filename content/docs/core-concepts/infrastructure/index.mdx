## Overview

ElizaOS infrastructure provides a comprehensive set of tools and services for building, testing, deploying, and managing AI agents. The infrastructure is designed to be developer-friendly, scalable, and secure.

## Core Components

### CLI Tools

The ElizaOS CLI (`@elizaos/cli`) is the primary interface for developers:

- **Project Management** - Create and manage projects, plugins, and agents
- **Development Tools** - Hot reload, auto-rebuild, and debugging
- **Testing Framework** - Unit, integration, and E2E testing
- **Deployment Utilities** - Build, package, and deploy to various platforms

[Learn more about CLI Tools →](/docs/core-concepts/infrastructure/cli-tools)

### Environment Configuration

Flexible environment management system:

- **Hierarchical Configuration** - Environment files with inheritance
- **Security Features** - Secret management and validation
- **Plugin Integration** - Per-plugin environment variables
- **Dynamic Updates** - Runtime configuration changes

[Learn more about Environment Configuration →](/docs/core-concepts/infrastructure/environment-config)

### Testing Infrastructure

Comprehensive testing framework:

- **Multiple Test Types** - Unit, integration, E2E, and shell tests
- **Test Utilities** - Port management, process control, timeouts
- **CI/CD Integration** - GitHub Actions and automated testing
- **Coverage Reports** - Code coverage analysis

[Learn more about Testing →](/docs/core-concepts/infrastructure/testing)

### Core Services

Essential services powering the platform:

- **File System Services** - Template management, directory operations
- **Process Management** - Server control, port handling
- **Plugin Services** - Discovery, loading, and registry
- **Documentation Tools** - Automated documentation generation

[Learn more about Services →](/docs/core-concepts/infrastructure/services)

## Development Workflow

### Quick Start

```bash
# Install CLI globally
bun install -g @elizaos/cli

# Create new project
elizaos create my-project
cd my-project

# Start development
elizaos dev
```

### Project Structure

```
project/
├── src/               # Source code
├── tests/             # Test files
├── .env              # Environment configuration
├── package.json      # Project configuration
└── tsconfig.json     # TypeScript configuration
```

### Development Commands

```bash
# Development mode with hot reload
elizaos dev

# Run tests
elizaos test

# Build for production
elizaos start --build

# Manage plugins
elizaos plugins list
elizaos plugins add @elizaos/plugin-openai
```

## Plugin Development

### Creating Plugins

```bash
# Create new plugin
elizaos create my-plugin --type plugin

# Develop plugin
cd plugin-my-plugin
elizaos dev

# Test plugin
elizaos test

# Publish plugin
elizaos publish
```

### Plugin Structure

```
plugin-name/
├── src/
│   └── index.ts      # Plugin definition
├── tests/            # Plugin tests
├── images/           # Required for registry
│   ├── logo.jpg      # 400x400px
│   └── banner.jpg    # 1280x640px
├── package.json      # With agentConfig
└── README.md         # Documentation
```

## Deployment Options

### Local Development

```bash
# Start local server
elizaos start

# With specific configuration
elizaos start --port 4000 --character agent.json
```

### Cloud Deployment

```bash
# Build for production
elizaos start --build

# Deploy to TEE
elizaos tee phala cvms create
```

### Container Deployment

```bash
# Build Docker image
docker build -t my-agent .

# Run container
docker run -p 3000:3000 my-agent
```
