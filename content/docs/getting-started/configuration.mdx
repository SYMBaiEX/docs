## Environment Configuration

### Interactive Environment Setup

The easiest way to configure your ElizaOS environment is through the interactive CLI:

```bash
# Interactive environment configuration (recommended)
elizaos env interactive

# Edit local environment variables
elizaos env edit-local

# List all environment variables
elizaos env list

# Check environment health
elizaos env check
```

### Manual Environment Setup

<Steps>
<Step>

### Create Environment File

```bash
# Copy example environment file
cp .env.example .env

# Edit with your preferred editor
nano .env
```

</Step>
<Step>

### Core Configuration

Add essential configuration variables:

```bash
# Required: Model Provider API Key
OPENAI_API_KEY=your_openai_key_here

# Optional: Alternative providers
ANTHROPIC_API_KEY=your_anthropic_key_here
GOOGLE_GENERATIVE_AI_API_KEY=your_google_key_here

# Database Configuration
POSTGRES_URL=postgresql://user:password@localhost:5432/eliza
# Or use PGLite for development
PGLITE_DATA_DIR=./data/pglite

# Server Configuration
SERVER_PORT=3000
SERVER_HOST=localhost
NODE_ENV=development
```

</Step>
<Step>

### Validate Configuration

```bash
# Test database connection
elizaos env test-db

# Test API keys
elizaos env test-keys

# Check overall health
elizaos env check
```

</Step>
</Steps>

## Project Types

ElizaOS v1.2.0 supports four distinct project types:

### Project Type Selection

<Tabs items={["Project", "Plugin", "Agent", "TEE"]}>
  <Tab value="Project">
    **Full ElizaOS Application** - Complete agent framework with all features - Web interface and
    API endpoints - Database integration and persistence - Plugin system support - Multi-platform
    connections ```bash elizaos create my-agent --type project ``` **Use when**: Building a complete
    AI agent application
  </Tab>
  <Tab value="Plugin">
    **Plugin Development** - Extend ElizaOS with custom functionality - Reusable components for the
    community - Action, provider, and evaluator development - Testing and publishing support ```bash
    elizaos create my-plugin --type plugin ``` **Use when**: Creating extensions for ElizaOS
  </Tab>
  <Tab value="Agent">
    **Agent Character Definition** - Standalone character JSON files - Personality and behavior
    configuration - Quick agent prototyping - Simple deployment scenarios ```bash elizaos create
    my-character --type agent ``` **Use when**: Defining agent personalities and behaviors
  </Tab>
  <Tab value="TEE">
    **Trusted Execution Environment** - Secure, decentralized agent deployment - Blockchain
    integration support - Privacy-preserving computations - Distributed agent networks ```bash
    elizaos create my-tee-agent --type tee ``` **Use when**: Deploying secure, decentralized agents
  </Tab>
</Tabs>

## Development Workflow

### Development Commands

<Tabs items={["Development", "Testing", "Deployment"]}>
  <Tab value="Development">
    ```bash # Start development server with hot-reload elizaos dev # Start with specific character
    elizaos dev --character characters/my-agent.json # Start with build elizaos dev --build # Start
    on custom port elizaos dev --port 8080 # Reconfigure services elizaos dev --configure ```
    **Development Features:** - Hot-reload on file changes - Detailed logging and debugging - Live
    configuration updates - Real-time error reporting
  </Tab>
  <Tab value="Testing">
    ```bash # Run all tests elizaos test # Run component tests elizaos test component # Run
    end-to-end tests elizaos test e2e # Run tests with coverage elizaos test --coverage # Run
    specific test file elizaos test --file my-test.spec.ts ``` **Testing Features:** - Unit and
    integration tests - Component testing framework - End-to-end testing support - Coverage
    reporting
  </Tab>
  <Tab value="Deployment">
    ```bash # Build for production elizaos start --build # Start production server elizaos start #
    Background process nohup elizaos start > elizaos.log 2>&1 & # With environment file elizaos
    start --env production ``` **Deployment Features:** - Production optimizations -
    Environment-specific configs - Background process support - Logging and monitoring
  </Tab>
</Tabs>

```bash

SERVER_HOST=localhost
NODE_ENV=development

## Best Practices

### Environment Setup

<Steps>
<Step>

### Use Environment Templates

Start with the provided `.env.example` file:

```bash
cp .env.example .env
```

</Step>
<Step>

### Secure API Keys

Never commit API keys to version control:

```bash
# Add to .gitignore
echo ".env" >> .gitignore
echo ".env.local" >> .gitignore
```

</Step>
<Step>

### Use Environment-Specific Files

Create separate environment files:

```bash
.env.development
.env.staging
.env.production
```

</Step>
<Step>

### Validate Configuration

Regularly check your configuration:

```bash
elizaos env check
```

</Step>
</Steps>

### Development Workflow

1. **Start with Interactive Setup**: Use `elizaos env interactive` for initial configuration
2. **Use Development Mode**: Run `elizaos dev` for active development
3. **Test Regularly**: Use `elizaos test` to catch issues early
4. **Keep Updated**: Run `elizaos update --check` regularly
5. **Monitor Logs**: Use appropriate `LOG_LEVEL` settings

### Plugin Management

1. **Choose Carefully**: Only install plugins you need
2. **Keep Updated**: Update plugins with `elizaos update --packages`
3. **Test Compatibility**: Test plugins after updates
4. **Monitor Performance**: Remove unused plugins

## Troubleshooting

### Common Configuration Issues

<Tabs items={["Environment", "Database", "Plugins", "CLI"]}>
  <Tab value="Environment">
    **API Key Issues** ```bash # Check if API key is loaded elizaos env list | grep OPENAI_API_KEY #
    Test API key validity elizaos env test-keys # Common issues: # - Extra spaces or quotes in .env
    # - Wrong API key format # - Insufficient API credits ``` **Environment Loading Issues** ```bash
    # Check .env file format cat .env | head -5 # Test environment loading node -e
    "console.log(process.env.OPENAI_API_KEY)" # Reset environment elizaos env reset ```
  </Tab>
  <Tab value="Database">
    **Database Connection Issues** ```bash # Test database connection elizaos env test-db # Check
    database URL elizaos env list | grep POSTGRES_URL # Switch to PGLite for development echo
    "PGLITE_DATA_DIR=./data/pglite" >> .env ``` **Database Migration Issues** ```bash # Reset
    database rm -rf ./data/pglite # Recreate database elizaos start --build ```
  </Tab>
  <Tab value="Plugins">
    **Plugin Installation Issues** ```bash # List installed plugins elizaos plugins
    installed-plugins # Remove problematic plugin elizaos plugins remove @elizaos/plugin-name #
    Update all plugins elizaos update --packages ``` **Plugin Compatibility Issues** ```bash # Check
    plugin versions elizaos plugins list --all # Install specific version elizaos plugins add
    @elizaos/plugin-name@1.2.0 ```
  </Tab>
  <Tab value="CLI">
    **CLI Issues** ```bash # Update CLI bun update -g @elizaos/cli # Check CLI version elizaos
    --version # Get help elizaos --help ``` **Permission Issues** ```bash # Fix permissions sudo
    chown -R $(whoami) ~/.bun # Or use sudo for global install sudo bun install -g @elizaos/cli ```
  </Tab>
</Tabs>
